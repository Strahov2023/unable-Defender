# Безопасная публикация Defender Control

Нельзя гарантировать отсутствие срабатываний Defender для приложения, которое временно меняет его настройки. Нельзя добавлять программе самоисключение, скрывать её поведение или обходить сканирование: это ухудшает безопасность и репутацию файла.

## Рекомендуемый выпуск

1. Соберите `DefenderControl.exe` командой `powershell -NoProfile -ExecutionPolicy RemoteSigned -File .\Build.ps1`.
2. Получите доверенный RSA-сертификат подписи кода или настройте Microsoft Trusted Signing.
3. Установите сертификат с закрытым ключом в `Cert:\CurrentUser\My`.
4. Подпишите оба файла: `powershell -NoProfile -File .\Sign-Release.ps1 -Thumbprint ВАШ_ОТПЕЧАТОК`.
5. Распространяйте вместе `DefenderControl.exe` и `DefenderControl.ps1`. Не изменяйте файлы после подписи.
6. Если Microsoft Defender классифицирует подписанный выпуск неверно, отправьте конкретные файлы как разработчик ПО через официальный портал: https://www.microsoft.com/wdsi/filesubmission

Smart App Control доверяет приложениям с RSA-подписью от доверенного поставщика. Самоподписанный сертификат полезен только для внутреннего тестирования и не создаёт публичную репутацию издателя.
